# Architecture Analysis Skill

## Overview

This skill helps you analyze, document, and improve system architecture by:
- Identifying architectural patterns and design decisions
- Creating architecture diagrams and documentation
- Evaluating trade-offs between different approaches
- Detecting architectural debt and improvement opportunities

## When to Use

- Reviewing system design before implementation
- Creating architecture decision records (ADRs)
- Evaluating monolith vs microservices trade-offs
- Documenting existing system architecture
- Identifying coupling and cohesion issues

## Capabilities

### 1. **Architecture Analysis**
- Identify architectural patterns (MVC, layered, event-driven, microservices)
- Analyze module boundaries and dependencies
- Evaluate coupling and cohesion metrics
- Detect architectural violations

### 2. **Documentation Generation**
- Create C4 model diagrams (Context, Container, Component, Code)
- Generate architecture decision records (ADRs)
- Document deployment topology
- Map data flow and integration points

### 3. **Design Evaluation**
- Compare architectural alternatives
- Assess scalability and extensibility
- Identify single points of failure
- Evaluate technology stack fit

## Output Formats

- Markdown architecture documentation
- Mermaid diagrams (flowchart, sequence, component)
- C4 model diagrams
- Architecture Decision Records (ADR)

## Best Practices

1. **Document decisions, not just structure**: Capture why, not just what
2. **Use multiple views**: Logical, physical, development, process
3. **Track evolution**: Architecture changes over time
4. **Validate assumptions**: Test architectural hypotheses early
