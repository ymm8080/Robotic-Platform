# 移动端操作手册 V1.0

> 适用版本：v3.5 | 更新日期：2026-07-05
> 适用平台：企业微信 / 飞书 / 钉钉

## 一、绑定流程

1. 打开 Web UI → 个人设置 → 移动端绑定
2. 选择绑定平台（微信/飞书/钉钉）
3. 扫码授权，完成 OAuth2 绑定
4. 系统发送测试消息，确认接收正常

> ⚠️ 未完成绑定的用户无法通过移动端执行任何操作。

## 二、告警接收

### 优先级分级

| 级别 | 说明 | 推送方式 | 提醒方式 |
|------|------|----------|----------|
| P0 | 急停/碰撞 | 全渠道即时推送 | 手机震动+响铃 |
| P1 | 订单异常 | 主渠道推送 | 手机震动 |
| P2 | 状态提醒 | 邮件+App消息 | 静默推送 |

### 告警卡片内容

每张告警卡片包含：
- 优先级标签 (P0/P1/P2)
- 告警标题
- 目标对象（机器人编号/订单号/区域编号）
- 告警详情
- 操作按钮（含操作对象编号）

## 三、操作规范

### 3.1 正常操作流程

1. 收到卡片后，**仔细阅读操作对象**（如"确认急停 R-01"）
2. 确认操作对象正确后，点击对应按钮
3. 危险操作（急停/取消订单/召回/区域封锁）会触发**二次确认卡片**
4. 在二次确认卡片中**再次核对操作对象**，点击"确认"或"取消"
5. 操作完成后，原卡片状态自动更新为"已处理"，无需重复点击

### 3.2 支持的移动端操作

| 操作 | 说明 | 需要二次确认 |
|------|------|-------------|
| 确认急停 | 停止指定机器人 | ✅ 是 |
| 确认召回 | 召回机器人回充电站 | ✅ 是 |
| 确认取消订单 | 取消指定订单 | ✅ 是 |
| 确认区域封锁 | 锁定指定区域 | ✅ 是 |
| 确认区域解锁 | 解锁指定区域 | ❌ 否 |
| 忽略 | 忽略此告警 | ❌ 否 |
| 查看详情 | 打开 Web UI 查看详情 | ❌ 否 |

### 3.3 六重校验机制

所有移动端操作经过六重校验：

1. **身份验证**：确认你的平台账号已绑定系统账号
2. **权限校验**：确认你有权执行该操作
3. **对象校验**：确认目标对象存在且状态允许操作
4. **防重放**：防止重复点击导致重复执行
5. **二次确认**：危险操作需二次确认
6. **执行前校验**：确认当前时刻可安全执行

> 校验失败时，卡片会返回拒绝原因。请根据原因进行相应操作。

## 四、禁止行为

- ❌ **禁止通过打字发送自然语言指令**（如"停掉机器人"）——系统不会执行
- ❌ **禁止转发操作卡片给他人**——操作权限与用户身份绑定
- ❌ **禁止在免打扰时段外忽略 P0 告警**——可能引发安全事故
- ❌ **禁止让他人代为操作**——审计日志追溯要求操作人一致
- ❌ **禁止在操作前不核对操作对象**——如"确认急停 R-01"不是"确认急停 R-10"

## 五、故障处理

### 收不到消息
1. 检查 Web UI 绑定状态是否正常
2. 重新扫码授权
3. 联系运维检查消息网关状态

### 点击按钮无响应
1. 等待 10 秒后刷新
2. 如仍无响应，前往 Web UI 操作
3. 不要重复点击（系统已做防重放处理）

### 误操作
1. **立即联系运维组长**
2. 通过 Web UI 撤销或补救
3. 误操作记录将保留在审计日志中

### 二次确认超时
- 二次确认卡片有效期为 **5 分钟**
- 超时后需重新点击原告警卡片发起操作

## 六、安全须知

- 所有移动端操作均记录在不可篡改的审计日志中，保留期 ≥3 年
- 关键操作（急停、取消订单）同时写入 WORM 存储
- 操作 IP 地址和 User-Agent 会被记录
- 异常操作行为（频繁失败、越权尝试）会触发安全告警

## 七、FAQ

**Q: 为什么我点击了按钮但操作没有执行？**
A: 可能原因：1) 六重校验中某层未通过；2) 操作对象状态不允许；3) 系统处于安全模式。请查看卡片返回的拒绝原因。

**Q: 可以同时收到多个平台的告警吗？**
A: 可以。P0 告警会同时推送到微信、飞书、钉钉、邮件四个渠道。你只需在一个平台操作即可。

**Q: 在飞书点击了确认，微信上的卡片会更新吗？**
A: 会。操作完成后，所有渠道的卡片状态会自动同步更新为"已处理"。

**Q: 如果消息网关宕机了怎么办？**
A: 消息网关独立部署，通过 Kafka 与核心业务主板解耦。网关宕机不影响调度引擎运行。告警会缓存在 Kafka 中，网关恢复后自动处理。在此期间请使用 Web UI 进行操作。
